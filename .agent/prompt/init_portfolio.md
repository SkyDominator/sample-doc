## 🐍 2. SDK 소스 코드 구현 (`sdk/nano_llm/engine.py`)

면접관들에게 익숙한 LLM 가속화 개념(Quantization, KV Cache)을 포함하고, 파이프라인이 파싱할 수 있도록 엄격한 Google 스타일 Docstring을 작성합니다.

```python
# sdk/nano_llm/engine.py
from typing import Generator, Optional
from .config import KVCacheConfig, QuantizationType

class NanoLLMEngine:
    """NanoLLMEngine은 가상 NPU 하드웨어 위에서 대규모 언어 모델(LLM)의 
    고속 가속 추론을 제어하는 메인 런타임 클래스입니다.
    """

    def __init__(self, model_path: str, quantization: QuantizationType = "FP16"):
        """
        Args:
            model_path (str): 컴파일된 모델 아티팩트의 경로.
            quantization (QuantizationType): 양자화 정밀도 설정 ('INT4', 'INT8', 'FP16').
        """
        self.model_path = model_path
        self.quantization = quantization
        self.is_loaded = False

    def load_to_device(self, device_id: int = 0) -> bool:
        """모델을 가상 NPU 하드웨어 메모리에 로드하고 추론 세션을 초기화합니다.

        Args:
            device_id (int): 모델을 바인딩할 하드웨어 디바이스 인덱스.
        Returns:
            bool: 가속기 로드 성공 여부.
        Raises:
            RuntimeError: 하드웨어(MOCK_RNGD_HARDWARE) 환경 변수가 없을 때 발생.
        """
        import os
        if not os.environ.get("MOCK_RNGD_HARDWARE"):
            raise RuntimeError("하드웨어 가속기 디바이스를 찾을 수 없습니다.")
        
        self.is_loaded = True
        return True

```

## 🤖 3. OpenAI API 기반 다국어 자동 번역 스크립트 (`scripts/translate_docs.py`)

JD의 **"Strong written English... translating complex systems behavior"** 요건을 자동화로 해결하는 핵심 카드입니다. 마크다운의 Frontmatter(설정값)와 코드 블록은 건드리지 않고 **설명 텍스트만 정확히 영어로 번역**합니다.

```python
# scripts/translate_docs.py
import os
import glob
from openai import OpenAI

client = OpenAI(api_key=os.environ.get("OPENAI_API_KEY"))

def translate_mdx_content(content: str) -> str:
    prompt = f"""
    You are an expert technical writer specializing in AI compilers and NPU hardware stacks.
    Translate the following Markdown/MDX content from Korean to professional technical English.
    
    CRITICAL RULES:
    1. Preserve all MDX frontmatter (e.g., title, description) keys, but translate their string values.
    2. DO NOT translate code blocks (```python ... ```) or inline code.
    3. Maintain all Markdown syntax structure exactly as is.
    
    Content to translate:
    {content}
    """
    
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.1
    )
    return response.choices[0].message.content

def translate_all_docs():
    # .mdx 파일 중 이미 번역된 .en.mdx가 아닌 원본 파일만 탐색
    mdx_files = [f for f in glob.glob("docs/content/docs/**/*.mdx", recursive=True) if not f.endswith(".en.mdx")]
    
    for file_path in mdx_files:
        en_path = file_path.replace(".mdx", ".en.mdx")
        
        # 원본 파일 읽기
        with open(file_path, "r", encoding="utf-8") as f:
            ko_content = f.read()
            
        print(f"🌐 Translating {file_path} to English...")
        en_content = translate_mdx_content(ko_content)
        
        # 영어 번역본 저장
        with open(en_path, "w", encoding="utf-8") as f:
            f.write(en_content)
            
    print("✅ 모든 문서의 OpenAI 다국어 자동 번역이 완료되었습니다!")

if __name__ == "__main__":
    translate_all_docs()

```

    - name: 빌드 및 변경사항 배포 (GitHub Pages 또는 Vercel)
      run: |
        pnpm --prefix docs install
        pnpm --prefix docs build
        # 배포 스크립트 실행...

```
