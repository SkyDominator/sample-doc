# NanoLLMEngine

`NanoLLMEngine` is a mock inference engine that imitates the RNGD device lifecycle.
It provides the basic flow from instance creation through device load, synchronous and streaming generation, memory queries, and session shutdown.

## Quickstart

#### Example

```python
import os

from nano_llm import InferenceConfig, NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = NanoLLMEngine(model_path="models/mock-llm.bin", quantization="FP16")
engine.load_to_device()

stream = engine.generate_streaming(
    "Summarize which settings the streaming output was generated from.",
    InferenceConfig(max_tokens=24, temperature=0.0),
)
for token in stream:
    print(token)

engine.unload()
```

## Constructor

```python
NanoLLMEngine(model_path, quantization='FP16', kv_cache_config=None)
```

### Parameters

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `model_path` | `str` | - | Path to the compiled mock model artifact. |
| `quantization` | `QuantizationType` | `'FP16'` | Quantization label reflected in the generation output prefix. |
| `kv_cache_config` | `KVCacheConfig \| None` | `None` | KV cache configuration object stored on the engine instance. |

## Usage Order

1. Create the instance with `NanoLLMEngine(...)`.
2. Use `load_to_device()` to open a mock device session and move the engine into a loaded state.
3. Use `generate()` to run a synchronous generation request.
4. Use `generate_streaming()` to consume the same generated result through a streaming interface.
5. Use `get_device_memory()` to inspect the current session's mock memory state.
6. Use `unload()` to close the session and reset the loaded state.

## Methods

### `load_to_device(device_id=0)`

Open a mock device session and mark the engine as loaded.

```python
NanoLLMEngine.load_to_device(device_id=0) -> bool
```

#### Parameters

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `device_id` | `int` | `0` | Mock hardware device index. |

#### Returns

True when the mock device session is ready.

#### Raises

| Error | When |
| :-- | :-- |
| `DeviceNotFoundError` | If ``MOCK_RNGD_HARDWARE`` is not set. |

#### Example

```python
import os

from nano_llm import NanoLLMEngine
from nano_llm.errors import DeviceNotFoundError

os.environ.pop("MOCK_RNGD_HARDWARE", None)

engine = NanoLLMEngine(model_path="models/mock-llm.bin")
try:
	engine.load_to_device(0)
except DeviceNotFoundError as exc:
	print(f"Expected error: {exc}")
```

### `generate(prompt, config=None)`

Run a synchronous mock generation request.

```python
NanoLLMEngine.generate(prompt, config=None) -> str
```

#### Parameters

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `prompt` | `str` | - | Input prompt text. |
| `config` | `InferenceConfig \| None` | `None` | Request configuration containing token budget and sampling controls. |

#### Returns

Mock output text prefixed with quantization and token-budget metadata.

#### Raises

| Error | When |
| :-- | :-- |
| `GenerationError` | If the engine has not been loaded yet. |

#### Example

```python
from nano_llm import InferenceConfig

cfg = InferenceConfig(batch_size=1, max_tokens=80, temperature=0.2)
out = engine.generate("Explain what the INT8 setting means for the memory usage label.", cfg)
print(out)
```

### `generate_streaming(prompt, config=None)`

Yield mock generation output through a streaming interface.

```python
NanoLLMEngine.generate_streaming(prompt, config=None) -> Generator[str, None, None]
```

#### Parameters

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `prompt` | `str` | - | Input prompt text. |
| `config` | `InferenceConfig \| None` | `None` | Request configuration containing token budget and sampling controls. |

#### Yields

Tokens produced by splitting the synchronous generation result on whitespace.

#### Raises

| Error | When |
| :-- | :-- |
| `GenerationError` | If the underlying generation request runs before the engine is loaded. |

#### Example

```python
from nano_llm import InferenceConfig

for token in engine.generate_streaming(
	"Explain how batched inference settings are reflected in streaming output.",
    InferenceConfig(max_tokens=40, temperature=0.1),
):
    print(token)
```

### `get_device_memory()`

Return mock memory telemetry for the current engine state.

```python
NanoLLMEngine.get_device_memory() -> dict[str, int]
```

#### Parameters

No parameters.

#### Returns

Dictionary with ``total_mb`` and ``used_mb`` values.

#### Example

```python
stats = engine.get_device_memory()
print(stats["total_mb"], stats["used_mb"])
```

### `unload()`

Close the mock device session and reset engine state.

```python
NanoLLMEngine.unload() -> bool
```

#### Parameters

No parameters.

#### Returns

True when the loaded state has been cleared.

#### Example

```python
engine.unload()
```

## Related types

### `InferenceConfig`

Configuration object passed to each mock generation request.

| Field | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `batch_size` | `int` | `1` | Number of prompts processed in one mock request. |
| `max_tokens` | `int` | `128` | Maximum token budget reflected in the output prefix. |
| `temperature` | `float` | `0.7` | Placeholder value representing sampling diversity. |

### `KVCacheConfig`

KV cache configuration object attached to the engine.

| Field | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `max_seq_len` | `int` | `4096` | Maximum sequence length the cache allocation is assumed to support. |
| `cache_dtype` | `str` | `'fp16'` | Precision label for KV cache storage. |

### `QuantizationType`

Supported literal values.

| Value |
| :-- |
| `INT4` |
| `INT8` |
| `FP16` |
| `FP32` |

## Errors

| Error | Raised by | Description |
| :-- | :-- | :-- |
| `DeviceNotFoundError` | `load_to_device()` | Raised when the mock hardware environment variable is missing. |
| `GenerationError` | `generate()`, `generate_streaming()` | Raised when generation is called on an engine that has not been loaded. |
