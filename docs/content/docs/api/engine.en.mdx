# RayKimLLMEngine

`RayKimLLMEngine` is a mock inference engine that imitates the RNGD device lifecycle.
It provides the basic flow from instance creation through device load, synchronous and streaming generation, memory queries, and session shutdown.

## Quickstart

#### Example

```python
import os
from raykim_llm import RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(model_path="models/mock-llm.bin")
print(engine.get_device_memory())

engine.load_to_device(0)
stats = engine.get_device_memory()
print(stats["total_mb"], stats["used_mb"])

engine.unload()
```

## Constructor

```text
RayKimLLMEngine(model_path, quantization='FP16', kv_cache_config=None)
```

### Parameters

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `model_path` | `str` | - | Path to the compiled mock model artifact. |
| `quantization` | `QuantizationType` | `'FP16'` | Quantization label reflected in the generation output prefix. |
| `kv_cache_config` | `KVCacheConfig | None` | `None` | KV cache configuration object stored on the engine instance. |

## Typical workflow

1. Create the instance with `RayKimLLMEngine(...)`.
2. `load_to_device()` - Open a mock device session and move the engine into a loaded state.
3. `generate()` - Run a synchronous mock generation request.
4. `generate_streaming()` - Yield the mock generation result through a streaming interface.
5. `get_device_memory()` - Return mock memory telemetry for the current engine state.
6. `unload()` - Close the mock device session and reset the engine state.

## Methods

### `load_to_device(device_id=0)`

Open a mock device session and move the engine into a loaded state.

```text
RayKimLLMEngine.load_to_device(device_id=0) -> bool
```

#### Parameters

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `device_id` | `int` | `0` | Mock hardware device index. |

#### Returns

Returns `True` when the mock device session is ready.

#### Raises

| Error | When |
| :-- | :-- |
| `DeviceNotFoundError` | Raised if `MOCK_RNGD_HARDWARE` is not set. |

#### Example

```python
import os
from raykim_llm import RayKimLLMEngine
from raykim_llm.errors import DeviceNotFoundError

os.environ.pop("MOCK_RNGD_HARDWARE", None)
engine = RayKimLLMEngine(model_path="models/mock-llm.bin")
try:
	engine.load_to_device(0)
except DeviceNotFoundError as exc:
	print(f"Expected error: {exc}")
```

### `generate(prompt, config=None)`

Run a synchronous mock generation request.

```text
RayKimLLMEngine.generate(prompt, config=None) -> str
```

#### Parameters

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `prompt` | `str` | - | Input prompt text. |
| `config` | `InferenceConfig | None` | `None` | Request configuration containing token budget and sampling controls. |

#### Returns

Returns a mock output string prefixed with quantization and token-budget metadata.

#### Raises

| Error | When |
| :-- | :-- |
| `GenerationError` | Raised when the engine has not been loaded yet. |

#### Example

```python
import os
from raykim_llm import InferenceConfig, RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(model_path="models/mock-llm.bin")
engine.load_to_device(0)

cfg = InferenceConfig(batch_size=1, max_tokens=80, temperature=0.2)
out = engine.generate("Explain what the INT8 setting means for the memory-usage label.", cfg)
print(out)
```

### `generate_streaming(prompt, config=None)`

Yield the mock generation result through a streaming interface.

```text
RayKimLLMEngine.generate_streaming(prompt, config=None) -> Generator[str, None, None]
```

#### Parameters

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `prompt` | `str` | - | Input prompt text. |
| `config` | `InferenceConfig | None` | `None` | Request configuration containing token budget and sampling controls. |

#### Yields

Yields tokens in order by splitting the synchronous generation result on whitespace.

#### Raises

| Error | When |
| :-- | :-- |
| `GenerationError` | Raised when the internal generation request runs before the engine is loaded. |

#### Example

```python
import os
from raykim_llm import InferenceConfig, RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(model_path="models/mock-llm.bin", quantization="INT8")
engine.load_to_device()

for token in engine.generate_streaming(
	"Explain how the token budget is reflected in the output format under the INT8 setting.",
	InferenceConfig(max_tokens=40, temperature=0.1),
):
	print(token)
```

### `get_device_memory()`

Return mock memory telemetry for the current engine state.

```text
RayKimLLMEngine.get_device_memory() -> dict[str, int]
```

#### Parameters

No parameters.

#### Returns

Returns a dictionary containing the `total_mb` and `used_mb` keys.

#### Example

```python
import os
from raykim_llm import RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(model_path="models/mock-llm.bin")
print(engine.get_device_memory())

engine.load_to_device(0)
stats = engine.get_device_memory()
print(stats["total_mb"], stats["used_mb"])

engine.unload()
```

### `unload()`

Close the mock device session and reset the engine state.

```text
RayKimLLMEngine.unload() -> bool
```

#### Parameters

No parameters.

#### Returns

Returns `True` when the loaded state has been cleared.

#### Example

```python
import os
from raykim_llm import RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(model_path="models/mock-llm.bin")
engine.load_to_device(0)
engine.unload()
```

## Related types

### `InferenceConfig`

Configuration object passed to each mock generation request.

| Field | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `batch_size` | `int` | `1` | Number of prompts processed together in one mock request. |
| `max_tokens` | `int` | `128` | Maximum token budget reflected in the output prefix. |
| `temperature` | `float` | `0.7` | Placeholder value representing sampling diversity. |

### `KVCacheConfig`

KV cache configuration object attached to the engine.

| Field | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `max_seq_len` | `int` | `4096` | Maximum sequence length the cache allocation is assumed to support. |
| `cache_dtype` | `str` | `'fp16'` | Precision label for KV cache storage. |

### `QuantizationType`

Supported literal values.

- `INT4`
- `INT8`
- `FP16`
- `FP32`

## Errors

| Error | Raised by | Description |
| :-- | :-- | :-- |
| `DeviceNotFoundError` | `load_to_device()` | Raised when the mock hardware environment variable is missing. |
| `GenerationError` | `generate()`, `generate_streaming()` | Raised when generation is called on an engine that has not been loaded. |
