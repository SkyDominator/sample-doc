# NanoLLMEngine

Mock inference engine for documentation and CI validation workflows.

## Quickstart

#### Example

```python
import os

from nano_llm import InferenceConfig, NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = NanoLLMEngine(model_path="models/mock-llm.bin", quantization="FP16")
engine.load_to_device()

stream = engine.generate_streaming(
	"스트리밍 출력의 목적을 설명해줘",
	InferenceConfig(max_tokens=24, temperature=0.0),
)
for token in stream:
	print(token)

engine.unload()
```

## Constructor

```python
NanoLLMEngine(model_path, quantization='FP16', kv_cache_config=None)
```

| Parameter | Type | Default | Description |
| :-- | :-- | :-- | :-- |
| `model_path` | `str` | - | Path to the compiled mock model artifact. |
| `quantization` | `QuantizationType` | `'FP16'` | Quantization label attached to mock generation output. |
| `kv_cache_config` | `KVCacheConfig | None` | `None` | Optional KV cache configuration stored on the engine. |

## Typical workflow

1. Create the instance with `NanoLLMEngine(...)`.
2. `load_to_device()` - Open a mock device session and mark the engine as loaded.
3. `generate()` - Run a synchronous mock generation request.
4. `generate_streaming()` - Yield mock generation output through a streaming interface.
5. `get_device_memory()` - Return mock memory telemetry for the current engine state.
6. `unload()` - Close the mock device session and reset engine state.

| Error | When |

#### Example

