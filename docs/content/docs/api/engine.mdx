# RayKimLLMEngine

`RayKimLLMEngine`는 RNGD 장치 수명주기를 흉내 내는 모의 추론 엔진입니다.
인스턴스 생성, 장치 로드, 동기/스트리밍 생성, 메모리 조회, 세션 해제까지의 기본 흐름을 제공합니다.

## 빠른 시작

#### 예제

```python
import os
from raykim_llm import InferenceConfig, RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(model_path="models/mock-llm.bin", quantization="INT8")
engine.load_to_device(0)

out = engine.generate("status", InferenceConfig(max_tokens=32))
tokens = list(engine.generate_streaming("status", InferenceConfig(max_tokens=32)))

print(out)
print(tokens)

engine.unload()
```

## 생성자

```text
RayKimLLMEngine(model_path, quantization='FP16', kv_cache_config=None)
```

### 매개변수

| 매개변수 | 타입 | 기본값 | 설명 |
| :-- | :-- | :-- | :-- |
| `model_path` | `str` | - | 컴파일된 mock 모델 아티팩트 경로입니다. |
| `quantization` | `QuantizationType` | `'FP16'` | 생성 결과 접두사에 반영되는 양자화 라벨입니다. |
| `kv_cache_config` | `KVCacheConfig | None` | `None` | 엔진 인스턴스에 보관할 KV cache 설정 객체입니다. |

## 사용 순서

1. `RayKimLLMEngine(...)`로 인스턴스를 생성합니다.
2. `load_to_device()` - 모의 장치 세션을 열고 엔진을 로드 상태로 전환합니다.
3. `generate()` - 동기 mock 생성 요청을 실행합니다.
4. `generate_streaming()` - mock 생성 결과를 스트리밍 인터페이스로 내보냅니다.
5. `get_device_memory()` - 현재 엔진 상태에 대한 mock 메모리 텔레메트리를 반환합니다.
6. `unload()` - 모의 장치 세션을 닫고 엔진 상태를 초기화합니다.

## 메서드

### `load_to_device(device_id=0)`

모의 장치 세션을 열고 엔진을 로드 상태로 전환합니다.

```text
RayKimLLMEngine.load_to_device(device_id=0) -> bool
```

#### 매개변수

| 매개변수 | 타입 | 기본값 | 설명 |
| :-- | :-- | :-- | :-- |
| `device_id` | `int` | `0` | 모의 하드웨어 장치 인덱스입니다. |

#### 반환값

모의 장치 세션이 준비되면 `True`를 반환합니다.

#### 예외

| Error | 조건 |
| :-- | :-- |
| `DeviceNotFoundError` | `MOCK_RNGD_HARDWARE` 환경 변수가 없으면 발생합니다. |

#### 예제

```python
import os
from raykim_llm import InferenceConfig, KVCacheConfig, RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(
    model_path="models/mock-llm.bin",
    quantization="INT8",
    kv_cache_config=KVCacheConfig(max_seq_len=8192, cache_dtype="fp16"),
)
engine.load_to_device(device_id=0)
```

### `generate(prompt, config=None)`

동기 mock 생성 요청을 실행합니다.

```text
RayKimLLMEngine.generate(prompt, config=None) -> str
```

#### 매개변수

| 매개변수 | 타입 | 기본값 | 설명 |
| :-- | :-- | :-- | :-- |
| `prompt` | `str` | - | 입력 프롬프트 문자열입니다. |
| `config` | `InferenceConfig | None` | `None` | 토큰 예산과 샘플링 제어를 담은 요청 설정입니다. |

#### 반환값

양자화 라벨과 토큰 예산 메타데이터를 앞에 붙인 mock 출력 문자열을 반환합니다.

#### 예외

| Error | 조건 |
| :-- | :-- |
| `GenerationError` | 엔진이 아직 로드되지 않은 상태에서 호출하면 발생합니다. |

#### 예제

```python
import os
from raykim_llm import RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(model_path="models/mock-llm.bin")
engine.load_to_device(0)
print(engine.generate("hello"))
```

### `generate_streaming(prompt, config=None)`

mock 생성 결과를 스트리밍 인터페이스로 내보냅니다.

```text
RayKimLLMEngine.generate_streaming(prompt, config=None) -> Generator[str, None, None]
```

#### 매개변수

| 매개변수 | 타입 | 기본값 | 설명 |
| :-- | :-- | :-- | :-- |
| `prompt` | `str` | - | 입력 프롬프트 문자열입니다. |
| `config` | `InferenceConfig | None` | `None` | 토큰 예산과 샘플링 제어를 담은 요청 설정입니다. |

#### 생성값

동기 생성 결과를 공백 기준으로 분리한 토큰들을 순서대로 생성합니다.

#### 예외

| Error | 조건 |
| :-- | :-- |
| `GenerationError` | 엔진 로드 전에 내부 생성 요청이 실행되면 발생합니다. |

#### 예제

```python
import os
from raykim_llm import InferenceConfig, RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(model_path="models/mock-llm.bin", quantization="INT8")
engine.load_to_device()

for token in engine.generate_streaming(
    "INT8 설정에서 토큰 예산이 출력 형식에 어떻게 반영되는지 설명해줘",
    InferenceConfig(max_tokens=40, temperature=0.1),
):
    print(token)
```

### `get_device_memory()`

현재 엔진 상태에 대한 mock 메모리 텔레메트리를 반환합니다.

```text
RayKimLLMEngine.get_device_memory() -> dict[str, int]
```

#### 매개변수

매개변수 없음.

#### 반환값

`total_mb`와 `used_mb` 키를 가진 사전을 반환합니다.

#### 예제

```python
import os
from raykim_llm import RayKimLLMEngine as Engine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = Engine(model_path="models/mock-llm.bin")
engine.load_to_device(0)
print(engine.is_loaded, engine.get_device_memory())

engine.unload()
print(engine.is_loaded)
```

### `unload()`

모의 장치 세션을 닫고 엔진 상태를 초기화합니다.

```text
RayKimLLMEngine.unload() -> bool
```

#### 매개변수

매개변수 없음.

#### 반환값

로드 상태가 해제되면 `True`를 반환합니다.

#### 예제

```python
import os
from raykim_llm import RayKimLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = RayKimLLMEngine(model_path="models/mock-llm.bin")
engine.load_to_device(0)
engine.unload()
```

## 관련 타입

### `InferenceConfig`

mock 생성 요청마다 전달하는 설정 객체입니다.

| 필드 | 타입 | 기본값 | 설명 |
| :-- | :-- | :-- | :-- |
| `batch_size` | `int` | `1` | 한 번의 mock 요청에서 함께 처리할 프롬프트 수입니다. |
| `max_tokens` | `int` | `128` | 출력 접두사에 반영되는 최대 토큰 예산입니다. |
| `temperature` | `float` | `0.7` | 샘플링 다양성을 나타내는 자리표시 값입니다. |

### `KVCacheConfig`

엔진에 부착되는 KV cache 설정 객체입니다.

| 필드 | 타입 | 기본값 | 설명 |
| :-- | :-- | :-- | :-- |
| `max_seq_len` | `int` | `4096` | cache 할당이 감당한다고 가정하는 최대 시퀀스 길이입니다. |
| `cache_dtype` | `str` | `'fp16'` | KV cache 저장 정밀도 라벨입니다. |

### `QuantizationType`

지원하는 리터럴 값입니다.

- `INT4`
- `INT8`
- `FP16`
- `FP32`

## 예외 타입

| Error | 발생 메서드 | 설명 |
| :-- | :-- | :-- |
| `DeviceNotFoundError` | `load_to_device()` | 모의 하드웨어 환경 변수가 없을 때 발생합니다. |
| `GenerationError` | `generate()`, `generate_streaming()` | 로드되지 않은 엔진에서 생성 호출이 실행될 때 발생합니다. |
