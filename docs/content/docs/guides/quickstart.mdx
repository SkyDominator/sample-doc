# 빠른 시작

이 가이드는 `nano-llm-engine` SDK로 모의 장치 세션을 열고 단일 생성 요청을 실행하는 최소 경로를 설명합니다.
실제 RNGD 하드웨어 대신 `MOCK_RNGD_HARDWARE` 환경 변수를 통해 장치 연동을 시뮬레이션합니다.

<Callout type="highlight" title="시작 전에 확인할 점">
`load_to_device()`를 호출하기 전에 `MOCK_RNGD_HARDWARE=true`를 설정해야 합니다. 이 샘플 런타임에서는 해당 환경 변수가 모의 장치 세션 시작 가능 여부를 결정합니다.
</Callout>

## 1. SDK 설치

프로젝트 루트에서 Python SDK를 설치합니다.

```bash
pip install -e ./sdk
```

	<Playground scenario="quickstart-sync-generate" />

	<TabsContent value="streaming">

	이 경로는 같은 mock 출력을 `generate_streaming()`이 반환하는 토큰 스트림 형태로 확인할 때 사용합니다.

	```python
	import os

	from nano_llm import InferenceConfig, NanoLLMEngine

	os.environ["MOCK_RNGD_HARDWARE"] = "true"
	engine = NanoLLMEngine(model_path="models/mock-llm.bin", quantization="FP16")
	engine.load_to_device()

	stream = engine.generate_streaming(
			"스트리밍 출력이 어떤 설정으로 생성됐는지 요약해줘",
			InferenceConfig(max_tokens=24, temperature=0.0),
	)
	for token in stream:
			print(token)

	engine.unload()
	```

## 3. 실패 시나리오 확인

하드웨어 환경 변수가 없으면 SDK가 즉시 오류를 발생시켜 장치 세션을 시작하기 전에 실행 환경 누락을 드러냅니다.

<Callout type="warning" title="의도된 실패 확인">
이 코드는 설정 검증이 제대로 동작하는지 확인할 때 사용합니다. 여기서 `DeviceNotFoundError`가 발생하는 것은 mock 엔진 고장이 아니라 보호 장치가 정상 동작한다는 뜻입니다.
</Callout>

```python
import os

from nano_llm import NanoLLMEngine
from nano_llm.errors import DeviceNotFoundError

os.environ.pop("MOCK_RNGD_HARDWARE", None)

engine = NanoLLMEngine(model_path="models/mock-llm.bin")
try:
	engine.load_to_device(0)
except DeviceNotFoundError as exc:
	print(f"예상된 오류: {exc}")
```
