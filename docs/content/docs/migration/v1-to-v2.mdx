# v1 to v2 마이그레이션

이 가이드는 최소 생성자 호출 중심의 v1 스타일에서, 엔진 설정과 요청 설정을 분리하는 v2 스타일로 옮길 때 확인할 포인트를 정리합니다.

## 변경 요약

- `quantization`은 기본값을 유지하되 필요한 경우 생성자에서 명시적으로 지정
- `KVCacheConfig`를 통해 캐시 관련 설정을 엔진 수준으로 분리
- `InferenceConfig`를 통해 요청별 토큰 예산과 샘플링 옵션을 전달
- 동일한 요청 설정을 동기/스트리밍 호출에 재사용 가능

## Before (v1 스타일)

```python
import os

from nano_llm import NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = NanoLLMEngine(model_path="models/mock-llm.bin")
engine.load_to_device(0)
print(engine.generate("hello"))
```

## After (v2 권장 스타일)

```python
import os

from nano_llm import InferenceConfig, KVCacheConfig, NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = NanoLLMEngine(
    model_path="models/mock-llm.bin",
    quantization="INT8",
    kv_cache_config=KVCacheConfig(max_seq_len=4096, cache_dtype="fp16"),
)
engine.load_to_device(0)
print(engine.generate("hello", InferenceConfig(max_tokens=32, temperature=0.0)))
```

## 단계별 이전 절차

1. `InferenceConfig`와 `KVCacheConfig` import를 추가합니다.
2. 엔진 생성 시 `kv_cache_config=KVCacheConfig(...)`를 전달합니다.
3. `generate()`와 `generate_streaming()`에 `InferenceConfig`를 넘깁니다.
4. 반환 문자열에서 `quantization`과 `max_tokens` 반영값을 확인합니다.

## 회귀 점검 포인트

- `MOCK_RNGD_HARDWARE` 없이 로드할 때 여전히 `DeviceNotFoundError`가 발생하는가?
- `engine.kv_cache_config`에 기대한 캐시 설정이 저장되는가?
- `generate_streaming()` 토큰을 다시 합치면 동기 출력과 같은 문자열이 되는가?
- `unload()` 후 로드 상태와 메모리 텔레메트리가 초기화되는가?
