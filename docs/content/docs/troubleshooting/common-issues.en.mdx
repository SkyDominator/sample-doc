# Common Issues

<Callout type="warning" title="Use this page by symptom first">
Start with the failure shape you can see right now: startup failure, state mismatch, or output surprise. Each tabbed section below groups related issues so you can compare causes without jumping around the page.
</Callout>

## Triage map

- [Startup and load failures](#1-startup-and-load-failures): Problems that occur before the runtime is ready to answer a generation request.
- [Memory and state mismatches](#2-memory-and-state-mismatches): Cases where the allocator state or reported device usage does not match what you expected.
- [Output contract surprises](#3-output-contract-surprises): Unexpected output prefixes or streaming behavior that still follow the current mock contract.

## 1) Startup and load failures

	<TabsContent value="missing-hardware">

	### DeviceNotFoundError: mock hardware not found

	Symptoms:
	- `load_to_device()` raises `DeviceNotFoundError`

	Cause:
	- The `MOCK_RNGD_HARDWARE` environment variable is not set.

	Resolution:

	```python
	import os

	os.environ["MOCK_RNGD_HARDWARE"] = "true"
	```

	</TabsContent>

	<TabsContent value="not-loaded">

	### GenerationError: engine is not loaded

	Symptoms:
	- `generate()` or `generate_streaming()` raises `GenerationError`

	Cause:
	- A generation function is called before `load_to_device()`.

	Resolution:

	```python
	import os
	from raykim_llm import RayKimLLMEngine

	os.environ["MOCK_RNGD_HARDWARE"] = "true"
	engine = RayKimLLMEngine(model_path="models/mock.bin")
	engine.load_to_device(0)
	print(engine.generate("ready"))
	```

	</TabsContent>
</Tabs>

## 2) Memory and state mismatches

<Tabs defaultValue="free-exceeds-used">
	<TabsList>
		<TabsTrigger value="free-exceeds-used">Free exceeds used</TabsTrigger>
		<TabsTrigger value="telemetry-zero">Telemetry stays at 0</TabsTrigger>
	</TabsList>

	<TabsContent value="free-exceeds-used">

	### RuntimeError: free size exceeds used memory

	Symptoms:
	- `DeviceMemory.free()` raises `RuntimeError`

	Cause:
	- The code attempts to release more bytes than are currently reserved.

	Resolution:

	```python
	from raykim_llm import DeviceMemory

	memory = DeviceMemory(total_bytes=1024 * 1024)
	memory.allocate(256 * 1024)

	try:
			memory.free(512 * 1024)
	except RuntimeError as exc:
			print(f"Expected error: {exc}")
	```

	Checking the `allocate()` return value together with `utilization()` makes it easier to understand the current state of the memory pool.

	</TabsContent>

	<TabsContent value="telemetry-zero">

	### `get_device_memory()` always returns 0

	Symptoms:
	- `used_mb` keeps showing `0`

	Cause:
	- `load_to_device()` has not been called yet, or `unload()` was already called.

	Resolution:

	```python
	import os
	from raykim_llm import RayKimLLMEngine as Engine

	os.environ["MOCK_RNGD_HARDWARE"] = "true"
	engine = Engine(model_path="models/mock.bin")
	print(engine.is_loaded)

	engine.load_to_device(0)
	print(engine.get_device_memory())
	```

	</TabsContent>
</Tabs>

## 3) Output contract surprises

<Tabs defaultValue="prefix">
	<TabsList>
		<TabsTrigger value="prefix">Unexpected prefix</TabsTrigger>
		<TabsTrigger value="streaming-shape">Streaming shape</TabsTrigger>
	</TabsList>

	<TabsContent value="prefix">

	### The output prefix is not what you expected

	Symptoms:
	- The result looks like `[mock:FP16:tokens=128] ...`, and the desired configuration is not visible.

	Cause:
	- `quantization` was not passed to the constructor, or `InferenceConfig(max_tokens=...)` was omitted.

	Resolution:

	```python
	import os
	from raykim_llm import InferenceConfig, RayKimLLMEngine

	os.environ["MOCK_RNGD_HARDWARE"] = "true"
	engine = RayKimLLMEngine(model_path="models/mock.bin", quantization="INT8")
	engine.load_to_device(0)
	print(engine.generate("status", InferenceConfig(max_tokens=32)))
	```

	</TabsContent>

	<TabsContent value="streaming-shape">

	### Streaming output does not look like fine-grained tokenization

	Symptoms:
	- `generate_streaming()` does not split into subword-like tokens.

	Cause:
	- The current implementation only splits the final string on whitespace.

	Resolution:

	```python
	import os
	from raykim_llm import RayKimLLMEngine

	os.environ["MOCK_RNGD_HARDWARE"] = "true"
	engine = RayKimLLMEngine(model_path="models/mock.bin")
	engine.load_to_device(0)

	tokens = list(engine.generate_streaming("stream test"))
	print(tokens)
	print(engine.generate("stream test"))
	```

	If you need the full string, call `generate()` directly. When checking the streaming path, validate it against the current contract of whitespace-based splitting.

	</TabsContent>
</Tabs>
