# Common Issues

## 1) DeviceNotFoundError: mock hardware not found

Symptoms:
- `load_to_device()` raises `DeviceNotFoundError`

Cause:
- The `MOCK_RNGD_HARDWARE` environment variable is not set.

Resolution:

```python
import os

os.environ["MOCK_RNGD_HARDWARE"] = "true"
```

## 2) GenerationError: engine is not loaded

Symptoms:
- `generate()` or `generate_streaming()` raises `GenerationError`

Cause:
- A generation function is called before `load_to_device()`.

Resolution:

```python
import os

from nano_llm import NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = NanoLLMEngine(model_path="models/mock.bin")
engine.load_to_device(0)
print(engine.generate("ready"))
```

## 3) RuntimeError: free size exceeds used memory

Symptoms:
- `DeviceMemory.free()` raises `RuntimeError`

Cause:
- The code attempts to release more bytes than are currently reserved.

Resolution:

```python
from nano_llm import DeviceMemory

memory = DeviceMemory(total_bytes=1024 * 1024)
memory.allocate(256 * 1024)

try:
	memory.free(512 * 1024)
except RuntimeError as exc:
	print(f"Expected error: {exc}")
```

Checking the `allocate()` return value together with `utilization()` makes it easier to understand the current state of the memory pool.

## 4) The output prefix is not what you expected

Symptoms:
- The result looks like `[mock:FP16:tokens=128] ...`, and the desired configuration is not visible.

Cause:
- `quantization` was not passed to the constructor, or `InferenceConfig(max_tokens=...)` was omitted.

Resolution:

```python
import os

from nano_llm import InferenceConfig, NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = NanoLLMEngine(model_path="models/mock.bin", quantization="INT8")
engine.load_to_device(0)
print(engine.generate("status", InferenceConfig(max_tokens=32)))
```

## 5) `get_device_memory()` always returns 0

Symptoms:
- `used_mb` keeps showing `0`

Cause:
- `load_to_device()` has not been called yet, or `unload()` was already called.

Resolution:

```python
import os

from nano_llm import NanoLLMEngine as Engine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = Engine(model_path="models/mock.bin")
print(engine.is_loaded)

engine.load_to_device(0)
print(engine.get_device_memory())
```

## 6) Streaming output does not look like fine-grained tokenization

Symptoms:
- `generate_streaming()` does not split into subword-like tokens.

Cause:
- The current implementation only splits the final string on whitespace.

Resolution:

```python
import os

from nano_llm import NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = NanoLLMEngine(model_path="models/mock.bin")
engine.load_to_device(0)

tokens = list(engine.generate_streaming("stream test"))
print(tokens)
print(engine.generate("stream test"))
```

If you need the full string, call `generate()` directly. When checking the streaming path, validate it against the current contract of whitespace-based splitting.
