# Inference Pipeline

This tutorial follows the flow of initializing a `NanoLLMEngine` instance, running generation and streaming requests, inspecting memory state and error paths, and then closing the session.

## 1) Initialize the Runtime

```python
import os

from nano_llm import InferenceConfig, KVCacheConfig, NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = NanoLLMEngine(
    model_path="models/mock-llm.bin",
    quantization="INT8",
    kv_cache_config=KVCacheConfig(max_seq_len=8192, cache_dtype="fp16"),
)
engine.load_to_device(device_id=0)
```

## 2) Run a Single Generation Request

```python
import os

from nano_llm import InferenceConfig, NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = NanoLLMEngine(model_path="models/mock-llm.bin", quantization="INT8")
engine.load_to_device(device_id=0)

cfg = InferenceConfig(batch_size=1, max_tokens=80, temperature=0.2)
out = engine.generate("Explain how the INT8 setting is displayed in the output string.", cfg)
print(out)

engine.unload()
```

`generate()` adds the `[mock:{quantization}:tokens={max_tokens}]` prefix so part of the current engine and request configuration is visible directly in the returned string.

## 3) Run a Streaming Request

```python
import os

from nano_llm import InferenceConfig, NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = NanoLLMEngine(model_path="models/mock-llm.bin", quantization="INT8")
engine.load_to_device(device_id=0)

for token in engine.generate_streaming(
    "Summarize the relationship between batched inference and latency.",
    InferenceConfig(max_tokens=40, temperature=0.1),
):
    print(token)

engine.unload()
```

`generate_streaming()` reuses the internal result from `generate()`, so joining the tokens back together yields the same payload as the synchronous call.

## 4) Check Memory Telemetry

```python
import os

from nano_llm import NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = NanoLLMEngine(model_path="models/mock-llm.bin")
print(engine.get_device_memory())

engine.load_to_device(0)
stats = engine.get_device_memory()
print(stats["total_mb"], stats["used_mb"])

engine.unload()
```

`get_device_memory()` reads the mock memory state currently held by the engine. Before load, `used_mb` is `0`; after load, it is `1024`.

## 5) Check Device Memory Pool Reservations

```python
from nano_llm import DeviceMemory

memory = DeviceMemory(total_bytes=32 * 1024 * 1024)
reserved = memory.allocate(16 * 1024 * 1024)
print(reserved, f"{memory.utilization():.2f}")

memory.free(16 * 1024 * 1024)
print(f"{memory.utilization():.2f}")
```

This step shows a pattern for checking reservation availability in an independent memory pool, separate from engine telemetry.

## 6) Error Handling Pattern

```python
import os

from nano_llm import NanoLLMEngine
from nano_llm.errors import DeviceNotFoundError

os.environ.pop("MOCK_RNGD_HARDWARE", None)

probe = NanoLLMEngine(model_path="models/mock-llm.bin")
try:
    probe.load_to_device(0)
except DeviceNotFoundError as exc:
    print(f"Expected error: {exc}")
```

## 7) Shutdown

```python
import os

from nano_llm import NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"

engine = NanoLLMEngine(model_path="models/mock-llm.bin")
engine.load_to_device(0)
engine.unload()
```

## Operational Checklist

1. Does calling `load_to_device()` without `MOCK_RNGD_HARDWARE` still raise `DeviceNotFoundError`?
2. After loading, does the `generate()` result reflect the current `quantization` and `max_tokens` values?
3. If you join `generate_streaming()` tokens back together, do you get the same string as the synchronous result?
4. After `unload()`, is `is_loaded` `False` and is `get_device_memory()["used_mb"]` equal to `0`?
