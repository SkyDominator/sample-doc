# 명시적 설정 객체 분리

이번 업데이트에서는 런타임 설정이 어느 계층에 속하는지 더 분명하게 드러나도록 설정 진입점을 정리했습니다.

## 변경 사항

- 프로세스 수준 설정은 `MOCK_RNGD_HARDWARE` 환경 변수로 유지됩니다.
- 엔진 수준 설정은 `RayKimLLMEngine(model_path, quantization, kv_cache_config)` 생성자에 모입니다.
- 요청 수준 설정은 `InferenceConfig(batch_size, max_tokens, temperature)`로 분리되어 각 호출마다 교체할 수 있습니다.
- `KVCacheConfig`가 별도 객체가 되면서 캐시 길이와 저장 정밀도를 엔진과 함께 명시적으로 보관할 수 있습니다.

## 영향

- 환경 준비, 엔진 구성, 요청 튜닝의 경계가 분리되어 호출 코드가 읽기 쉬워졌습니다.
- 하나의 엔진 인스턴스를 유지한 채 요청별로 `max_tokens`나 `temperature`만 바꾸는 패턴을 명확하게 지원합니다.
- `kv_cache_config`가 엔진 속성으로 저장되므로 캐시 설정을 나중에 다시 확인하기 쉽습니다.

## 예시

```python
from raykim_llm import InferenceConfig, KVCacheConfig, RayKimLLMEngine

engine = RayKimLLMEngine(
	model_path="models/mock-llm.bin",
	quantization="INT8",
	kv_cache_config=KVCacheConfig(max_seq_len=8192, cache_dtype="fp16"),
)

cfg = InferenceConfig(batch_size=2, max_tokens=64, temperature=0.2)
print(engine.kv_cache_config.max_seq_len, cfg.max_tokens)
```
