# Generation output metadata added

This version fixes the mock output format so you can see immediately which configuration produced a result.

## What changed

- `generate()` always returns a result with a prefix containing the quantization label and token budget.
- `InferenceConfig.max_tokens` is reflected directly in the output string, so request settings are easier to trace.
- `generate_streaming()` does not create a separate format and instead splits the same generated result on whitespace.

## Output contract

```text
[mock:{quantization}:tokens={max_tokens}] {prompt}
```

- `quantization` comes from the engine constructor.
- `max_tokens` comes from `InferenceConfig`.
- `{prompt}` preserves the prompt string exactly as passed at call time.
- `batch_size` and `temperature` are stored on the request object but are not included in the current output prefix.

## Impact

- Because the synchronous and streaming paths share the same payload, it is easy to compare their behavior.
- In tests, checking only the string prefix is enough to verify the active quantization setting and token-budget reflection.

## Example

```python
import os
from nano_llm import InferenceConfig, NanoLLMEngine

os.environ["MOCK_RNGD_HARDWARE"] = "true"
engine = NanoLLMEngine(model_path="models/mock-llm.bin", quantization="INT8")
engine.load_to_device(0)

out = engine.generate("status", InferenceConfig(max_tokens=32))
tokens = list(engine.generate_streaming("status", InferenceConfig(max_tokens=32)))

print(out)
print(tokens)

engine.unload()
```
